## 1. Суть (слова Pavel'а, не усреднять)

Сигнатурное лицо — максимально пиздатое, но лёгкое для быстрого рендера и липсинка. Основная задача — **максимальный баланс посередине: пиздатость ↔ скорость. Цена пиздатости измеряется скоростью.** С ним можно поговорить голосом, задержки минимальные, и главная новая фишка — **перебивание**: эффект как с человеком говоришь — переспросил, перебил, он быстро отвечает.

## 2. Хард-правила (нерушимые)

1. Липсинк в реалтайме.
2. Перебивание голосом — центральная фишка (§5).
3. Латентность **≤ 1 секунды** от «пользователь замолчал» до «Джарви начал отвечать». Потолок, не цель.
4. **Всё бесплатно**: хостинг, STT, LLM, TTS, рендер, туннели. 0₽.
5. При дефиците скорости жертвовать в порядке: **микрожизнь первой** → шейдеры/свет → … голос и виземы держать до последнего.
6. Может быть тупой по содержанию — но мгновенной и живой.
7. Клонирование голоса реального актёра — запрещено. Архетип/манера — можно.

## 4. Архитектура (рекомендованная; менять можно при веских причинах, хард-правила — нельзя)

- **Лицо:** рендерит браузер зрителя (WebGL/R3F). Качественная стилизованная 3D-голова с blendshapes/виземным ригом. Пиздатость добывается ригом, светом, шейдерами, bloom/rim — НЕ полигонами и НЕ фотореал-нейронками (MuseTalk-путь законсервирован: убивает ≤1с). Фотореал-стрим с карты Pavel'а — зарезервирован как v2-эксперимент, архитектура не должна его блокировать.
- **STT:** Web Speech API в браузере зрителя (Chrome, русский, interim-результаты → триггер барж-ина).
- **Мозг:** Cerebras. ⚠️ ПРОВЕРЕНО: хаб Aeza `:8090` — pass-through, ключ НЕ держит (форвардит Authorization клиента) и работает по голому HTTP → статика с github.io (HTTPS) его дёргать не может (mixed content блокируется молча) и ключ светился бы. Поэтому: **развернуть на Aeza отдельный микро-прокси на новом порту** (портировать логику `docs/worker/head-proxy.js`: инжект ключа из `jarvis/litellm/.env`, CORS origin-allowlist, per-IP rate limit ~10 req/мин, кап `max_tokens`~200, системный промпт зашит на сервере, `Access-Control-Max-Age`). Конфиг `:8090` НЕ трогать — на нём прод-лэйны. **HTTPS наружу — trycloudflare quick tunnel** (`cloudflared tunnel --url` — бесплатно, без аккаунта) под systemd; URL эфемерный → клиент читает эндпоинт из `config.json` в репо сайта, скрипт на Aeza при смене URL сам правит и пушит. Запасной стабильный путь: caddy + `91-108-241-13.sslip.io` (Let's Encrypt без домена). Cloudflare Worker — резерв (нужен аккаунт Pavel'а, ночью недоступен). Первым делом дёрнуть `/v1/models` — взять живую модель (llama-3.3-70b из воркера могла протухнуть).
- **Голос:** bake-off двух бесплатных кандидатов, переключение **одним параметром**:
  1. edge-tts `ru-RU-Dmitry` — живой, отдаёт word boundaries (почти готовые тайминги для визем); минус — ходит в сеть Microsoft;
  2. Silero v4 ru — полностью локальный на машине Pavel'а; тайминги визем добывать самим (фонемизатор/распределение по длительности).
  Резерв в том же параметре: женский мягко-деловой. Выбор ушами Pavel'а утром.
- **Машина Pavel'а** (12 ГБ VRAM, всегда включена, любые мощности в распоряжении) = голосовой сервер для Silero-варианта; наружу — бесплатный туннель. Если edge-tts выигрывает bake-off — TTS можно держать на Aeza, GPU не нужен.
- **Стриминг — ключ к латентности:** Джарви озвучивает первое предложение, пока LLM дописывает остальное. ⚠️ ОБЯЗАТЕЛЬНЫЙ ПРИЁМ: НЕ ждать `isFinal` от SpeechRecognition — Chrome финализирует фразу через ~500–900 мс тишины и это не настраивается, бюджет ≤1с умирает. Свой endpointing по interim: interim-текст не менялся ~250–400 мс → фраза закончена, слать в LLM сразу (поздний final — игнор/докоррекция). С этим приёмом бюджет: ~300 мс endpointing + ~250–350 мс мозг (TTFT) + первый кусок TTS ≈ 0.6–0.9 с. Замерить вживую и доказать ≤1с.

## 5. Перебивание — сердце фишки

1. Стоп **<50 мс**, обрыв на полуслове (не дослушивая фразу до конца).
2. В историю диалога попадает **только произнесённая часть** ответа, не весь сгенерированный текст. «Что-что ты сказал?» — повторяет именно оборванное место; «продолжай» — продолжает с обрыва, не заново. ⚠️ Реализация: говорить **по предложениям отдельными utterance** — гранулярность истории = завершённые предложения (utterance кончился = произнесено). На `onboundary` не ставить: у сетевого «Google русский» он НЕ стреляет; где стреляет — бонус-уточнение. Почленение по предложениям заодно обходит Chrome-баг замолкания длинных utterance (~15с).
3. В момент перебивания голова видимо переходит в состояние «слушаю» (это реакция-состояние, обязательная; не путать с микрожизнью, которая первая под нож).
4. **Защита от самоперебивания**: голос Джарви из колонок попадает в микрофон и распознавалка его честно транскрибирует. ⚠️ На захват микрофона SpeechRecognition повлиять НЕЛЬЗЯ (constraints у него нет), а AEC не вычитает локальные SAPI-голоса (озвучка идёт мимо аудиотракта Chrome — опорного сигнала нет). Первая линия обороны — **текстовый фильтр**: нормализованное fuzzy-сравнение interim-результата с текстом, который Джарви сейчас произносит (текст текущего utterance известен); совпало → игнор; не совпало и ≥2 слов → человек, барж-ин. Одно короткое слово — не триггер. Наушники — аварийный демо-фоллбэк. Без этой защиты зациклится на первом же ответе.
5. v2 (после этажа Б): «ага/угу» поверх речи = поддакивание, не перебивание.
6. Реплики Джарви — **1–2 предложения** (закрепить системным промптом). Это часть конструкции перебивания и латентности, не стилистика.

## 6. Личность

- Имя: **Джарви** (решение Pavel'а; «Джарвис» не называть).
- Архетип: учтивый ИИ-дворецкий — бархатная подача, спокойствие, сухая ирония, никогда не суетится, краткость. Темп речи чуть ниже среднего, паузы.
- Голос мужской (см. bake-off §4); женский мягко-деловой — резерв.
- Аудитория: деловые люди (демо-сайт полуприватный, зрители штучные), но возможны дизайнеры/школы — тон универсальный мягко-деловой, без подобострастия.
- Системный промпт написать исполнителю; русский язык основной.

## 7. Размещение, этажи, дедлайн

- **Место:** страница AI-BOTS, **первое окошко-ссылка** — Джарви живёт там. Запасной вариант (если интеграция в AI-BOTS встанет): отдельная страница с ним, но опубликованная.
- **Этаж А — обязательный к утру, без отговорок:** Джарви на опубликованном сайте: голова, слушает, отвечает, **перебивается** — полный разговорный контур. Голос допускается временный браузерный `speechSynthesis` (cancel реально <50 мс — проверено, для барж-ина годен). ⚠️ Липсинк этажа А: амплитудный анализ от `speechSynthesis` НЕВОЗМОЖЕН (он не отдаёт аудиопоток, AnalyserNode подключить не к чему — схема из SIGNATURE_HEAD §7 для этажа А не работает) → процедурный рот: шевелится пока utterance активен, `onboundary` как акценты где стреляет. Настоящая амплитуда/виземы — этаж Б (`<audio>` → MediaElementSource → AnalyserNode).
- **Стартовая кнопка обязательна** («Поговорить с Джарви»): Chrome не даст ни `speechSynthesis.speak()`, ни микрофон без user activation — «открыл страницу и Джарви уже слушает» невозможно. В обработчике кнопки разом: getUserMedia-разрешение, разблокировка speechSynthesis, resume AudioContext, старт recognition.
- **Этаж Б — догоняет (ночью если успеешь, иначе следующим заходом):** живой голос (bake-off §4), виземный липсинк вместо амплитудной челюсти, шейдерная полировка.
- **Публикация:** закончил — сразу push (GitHub Pages автодеплой), не спрашивая. Проверять на прод-билде.
