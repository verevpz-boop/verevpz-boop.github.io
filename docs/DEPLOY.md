# Публикация сайта (GitHub Pages)

Всё готово к деплою. Осталось то, что может сделать только Pavel (вход в свой GitHub).

## Важно про приватность и страх «взлома»

Твой сайт — **статика**: HTML/CSS/JS + 3D-модели. Нет бэкенда, базы, логинов, форм, ключей. Взламывать нечего. То, что публикуется (папка `out/`) — это ровно те файлы, которые браузер и так скачивает у любого посетителя.

## Нюанс GitHub Pages (надо знать)

- **Публичный репозиторий** → Pages бесплатны. Код виден, но кода-секрета у нас нет.
- **Приватный репозиторий** → Pages работают только на платном GitHub Pro (~$4/мес).

**Рекомендация:** сделать **публичный** репо `<твой-логин>.github.io` и в нём только этот сайт. Бояться нечего — секретов в коде нет (проверено: ни `.env`, ни ключей).

## Шаги (5 минут, утром)

1. Создай репозиторий на github.com (название: `<логин>.github.io` для корневого адреса, либо любое — тогда адрес будет `.../имя-репо/`).
2. В терминале из `D:\pavel-site`:
   ```
   git remote add origin https://github.com/<логин>/<репо>.git
   git branch -M main
   git push -u origin main
   ```
3. На github.com → репо → **Settings → Pages → Source: GitHub Actions**.
4. Через ~2 минуты сайт живой. Ссылка появится во вкладке **Actions** и в Settings → Pages.

## Если репо НЕ `<логин>.github.io`, а с именем (напр. `pavel-site`)

Адрес будет `https://<логин>.github.io/pavel-site/`. Тогда в `next.config.ts` добавь:
```ts
basePath: "/pavel-site",
```
и снова `git push`. (Если репо `<логин>.github.io` — basePath не нужен.)

## Дальше

Каждый `git push` → сайт автоматически пересобирается и обновляется (workflow `.github/workflows/deploy.yml`). Песочница — локально на `localhost:3000`, как накопится — пушим.
